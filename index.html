Contract as Test

Tests for Free - Specmatic parses your API Specification files into Test Scenarios and then generates requests and fires it at your application. Once it response is received it verifies status, schema, headers, etc. based your API Specification. All of this with a “Zero Code” approach. Spematic generates two types of tests.

Positive Tests Averify if the API implementation meets basic criterea as per your API Spefication

Negative Tests verify boundary conditions to identify error handling, unexpected behavior, etc.

Learn More

Contract as Stub

A component test setup should be able to isolate the “System Under Test” from its dependencies. Smart Service Virtuatlization enables you to achieve this by truly emulating the dependencies.

Expectation Validation ensures that the responses that are setup in Specmatic Service Virtualization Server are continuously evaluated against the API Specification of the Provider it is emulating.

Dynamic and Static Stubbing allows on the fly expectations setup over Specmatic’s http interface as well as at startup respectively. Both modes go through the above validation.

Learn More

Backward Compatibility

As API Specifcation authors / API designers it is important for us understand if our changes to existing APIs can break compatibility with existing API Consumers / Clients. Specmatic can “Contract vs Contract” testing between two API Specification files and provide accurate results about compatibility issues. All this without even you having to implement the changes in API Implementation / Provider or API Client / Consumer.

Learn More

Treat Contract as Code

Specmatic and Contract Driven Development promotes treating your API Specifications as code and storing them in a Version Control System where they truly belong.

Central Contract Repo - The “Single Source of Truth” for all your API Specifications so that stakeholders across the organization are on the same page.

Pull / Merge Request based Collaboration - Specmatic can help you identify backward compatibility breaking changes automatically as part of your pre-merge checks so that you can focus on API Design.
