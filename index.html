API Specifications as Executable Contracts

Specmatic supercharges your API Specifications by leveraging them as “Executable Contracts”. Specmatic does this by

  • Smart Service Virtualization

    (Emulating the Provider / Service based on the API Spefication) which allows Consumer / Client Applications to be Developed and Tested indepent of the Provider

  • Contract as Test

    (Emulating the Consumer / Client applicatin for the Provider) generates Test Scenarios based on the API Specification File without you have to write any code

Contract as Test

Tests for Free - Specmatic parses your API Specification files into Test Scenarios and then generates requests and fires it at your application. Once it response is received it verifies status, schema, headers, etc. based your API Specification. All of this with a “Zero Code” approach. Spematic generates two types of tests.

  • Positive Tests

    verify if the impleme meets basic criterea as per your API Spefication

  • Negative Tests

    verify the boundary conditions to identify issues related to areas such as error handling

Contract as Tests (Smart Service Virtualization)

What is so smart about Specmatic’s Service Virtualization?

  • Expectation Validation

    ensures that the responses that are setup in Specmatic Service Virtualization Server are continuously evaluated against the API Specification of the Provider it is emulating.

  • Dynamic and Static Stubbing

    allows on the fly expectations setup over Specmatic’s http interface as well as at startup respectively. Both modes go through the above validation

Backward Compatibility

As API Specifcation authors / API designers it is important for us understand if our changes to existing APIs can break compatibility with existing API Consumers / Clients. Specmatic can “Contract vs Contract” testing between two API Specification files and provide accurate results about compatibility issues. All this without even you having to implement the changes in API Implementation / Provider or API Client / Consumer.
